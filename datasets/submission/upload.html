J. Upload to BigPicture

Encrypt and submit your validated bundle to the NBIS staging inbox with bp-submitter. This is the final step of the operator journey.

This is a staging (test) upload

The A–J manual is a test submission, so this step uploads to BigPicture’s staging environment — the s3cmd-inbox.conf NBIS provides for the manual points there. The signed Terms of Use form (internal) is not needed for a staging test — you only sign and send it to submit@bigpicture.eu before a production submission.

Precondition: NBIS receiver credentials

NBIS provides two files before you can upload. Because the A–J manual is a staging test, place both in your staging credentials directory (PIPELINE_SUBMITTER_CREDS_DIR_STAGING, default certs/staging) — a subfolder of the certs/ directory you set up in F. TLS + credentials. For a real production submission the equivalent files go in PIPELINE_SUBMITTER_CREDS_DIR_PRODUCTION (default certs/production) instead, so the two instances never share credentials:

File Purpose
s3cmd-inbox.conf S3 connection configuration for the NBIS inbox
crypt4gh_bp_key.pub NBIS receiver public key for encryption

Contact NBIS if you do not have these files. Do not commit them to version control — they are already covered by the .gitignore.

1. Dry run

Run a dry run first. It validates your credentials, bundle path, and encryption readiness without transferring any data.

  • Windows
  • macOS
  • Linux

In Git Bash, run:

make upload-dry-run DATASET=<project_dir>
make upload-dry-run DATASET=<project_dir>
make upload-dry-run DATASET=<project_dir>

Replace <project_dir> with the project directory name — it is the value after Project: in the make export-check output from H. Run a submission, for example DATASET_test-submission.10494c10-0d6b-4103-89e0-4909ad6e23af.

make upload-dry-run defaults to staging. To dry-run against production credentials instead, add TARGET=production.

Verify

A passing dry run starts with:

Dry-run upload of '<project_dir>' to staging (zero bytes) ...

and exits 0 with no ERROR lines. Fix any reported errors before uploading.

2. Upload

When the dry run passes, run the real upload. make upload requires a TARGET — it never guesses which instance to push to. The A–J manual is a staging test, so use TARGET=staging:

  • Windows
  • macOS
  • Linux

In Git Bash, run:

make upload DATASET=<project_dir> TARGET=staging
make upload DATASET=<project_dir> TARGET=staging
make upload DATASET=<project_dir> TARGET=staging

For a real submission, use TARGET=production instead — it reads the separate PIPELINE_SUBMITTER_CREDS_DIR_PRODUCTION credentials:

make upload DATASET=<project_dir> TARGET=production
Use --threads 1 (the default)

NBIS has explicitly asked submitters not to use parallel upload threads. make upload enforces --threads 1 by default.

If NBIS has explicitly told you that more threads are allowed for your site, you can override:

make upload DATASET=<project_dir> TARGET=staging THREADS=4

Notify NBIS

Once your submission is completed, notify NBIS by sending an email. The subject of the email should be in the format of: submission - DATASET_{IDENTIFIER}, for example submission - DATASET_test-submission.10494c10-0d6b-4103-89e0-4909ad6e23af.

The email should contain:

  • Name of the uploader
  • Name of the dataset

Resumable uploads

bp-submitter tracks progress in a local SQLite database at ${PIPELINE_SUBMITTER_STATE}/upload.db (default: $PIPELINE_ROOT/submitter-state/upload.db). If an upload is interrupted, re-run the same make upload DATASET=<project_dir> TARGET=<staging|production> command — it resumes from where it left off.

To start over (for example, after replacing a corrected bundle), delete upload.db from your PIPELINE_SUBMITTER_STATE directory before re-running.

If something goes wrong

  • TARGET is required: ... — make upload will not run without a target. Re-run with TARGET=staging (the manual) or TARGET=production.
  • Invalid TARGET '<x>': expected 'staging' or 'production' — the TARGET value is mistyped. Use exactly staging or production.
  • Missing creds file: .../s3cmd-inbox.conf or Missing creds file: .../crypt4gh_bp_key.pub — one of the NBIS-provided files is absent from the credentials directory for the TARGET you chose (certs/staging for TARGET=staging, certs/production for TARGET=production). Place the missing file in that directory and retry.
  • DATASET not set — re-run with DATASET=<project_dir>.
  • Dataset dir not found — the project directory name does not match what is under PIPELINE_OUTBOX_DIR. Confirm the name from make export-check output (step H).
  • Connection or authentication errors — your s3cmd-inbox.conf may be expired or misconfigured. Contact NBIS for an updated file.
  • Upload stalls and does not resume — delete upload.db from PIPELINE_SUBMITTER_STATE and retry from the beginning.
